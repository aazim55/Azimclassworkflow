GIT WORKFLOW

Git workflow — это правила, которые помогают в командной работе организовать работу с кодом. Он показывает как создавать ветки и объединять изменения, чтобы все могли наблюдать за изменениями в проекте и что происходит в проекте. Gitworlflow нужен для того чтобы сделать командную работу более проще и понятней для остальных участников проекта и избежать каши в коде

GIT REBASE

Git rebase — это команда, которая позволяет переносить коммиты из одной ветки в другую, создавая историю в линию. Это делает историю более чистой, без слияний. Используется для обновления ветки с последними изменениями основной ветки.

Отличия git rebase и git merge:

  • Git rebase
  • Создает линейную историю.
  • Переносит коммиты на вершину другой ветки.
  • Конфликты решаются в процессе.
  • Git merge
  • Сохраняет историю слияний.
  • Создает новый коммит слияния.
  • Проще в использовании для объединения веток.

GIT FETCH

Git fetch — команда, которая загружает изменения из удаленного репозитория, обновляя информацию о ветках, но не необьединяет их с твоей основной веткой. Поэтому мы можем увидеть новые коммиты перед объединением.

Отличия git fetch и git pull:

  • Git fetch
  • Загружает изменения из удаленного репозитория.
  • Не меняет твою jcyjdye. ветку.
  • Git pull
  • Загружает и сразу сливает изменения с твоей локальной веткой.
  • Может привести к непоняткам в команде
